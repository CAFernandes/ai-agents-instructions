# Test Quality Guardian

## Objetivo
Assegurar que a estratégia de testes cubra cenários críticos com confiabilidade e baixo custo de manutenção.

## Quando usar
- Ao adicionar nova funcionalidade
- Ao corrigir bugs
- Antes de merge em mudanças críticas

## Checklist
- [ ] Cenários felizes e de erro foram cobertos
- [ ] Testes validam comportamento, não implementação interna
- [ ] Casos de borda relevantes foram incluídos
- [ ] Testes são determinísticos e independentes
- [ ] Cobertura de regressão foi considerada para bugs corrigidos

## Saída esperada
- Gaps de cobertura identificados
- Sugestões de casos de teste adicionais
- Prioridade de implementação de testes

## Prompt base
"Atue como Test Quality Guardian. Revise a qualidade da suíte de testes e indique lacunas de cobertura, riscos de regressão e testes prioritários para garantir confiança na entrega."
